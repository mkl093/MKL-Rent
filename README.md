# rental-inventory

Система учёта оборудования для AV-рентала (MVP). Модульный монолит на FastAPI +
PostgreSQL + SQLAlchemy 2 + Jinja2 + HTMX + Bootstrap 5. Полное ТЗ — в
[`TECHNICAL_SPECIFICATION.md`](TECHNICAL_SPECIFICATION.md).

## Прогресс по этапам (ТЗ §44)

- [x] **Этап 1. Основа** — каркас, конфигурация, БД/миграции, авторизация (сессии,
  CSRF, rate-limit), layout, настройки компании, тестовая инфраструктура.
- [x] **Этап 2. Склад** — категории/подкатегории, модели (количественные и
  посерийные), экземпляры со штрих-кодами, остатки и истории, статусы, кейсы/рэки,
  фото (Pillow), фильтры/сортировка, глобальный поиск по штрих-коду, архив.
- [x] **Этап 3. Проекты и доступность** — проекты, нумерация `PRJ/EST/PL-YYYY-NNN`,
  даты с пересечениями, движок доступности и дефицита, бронирование со статус-машиной,
  копирование, архив, виджеты дашборда.
- [ ] **Этап 4. Смета** — выбор оборудования, цены/коэффициенты, VAT/скидка, произвольные строки.
- [ ] **Этап 5. Packing-лист** · **Этап 6. Штрих-коды/mobile** · **Этап 7. PDF** · **Этап 8. Backup/hardening**.

## Требования

- Python 3.12+
- PostgreSQL (локально запущенный или доступный по сети) — для боевого режима и миграций.

## Локальный запуск (venv)

```bash
# 1. Зависимости (в активированном .venv)
pip install -e ".[dev]"

# 2. Конфигурация
cp .env.example .env
# отредактируйте .env: APP_SECRET_KEY и доступ к PostgreSQL (DATABASE_HOST=localhost)
# сгенерировать секрет: python -c "import secrets; print(secrets.token_urlsafe(48))"

# 3. Миграции (создают/обновляют схему)
alembic upgrade head

# 4. Первый пользователь
python -m scripts.create_user --username admin

# 5. Запуск
uvicorn app.main:app --reload      # либо: python main.py
```

Откройте http://localhost:8000/ → форма входа. После входа доступны разделы
**Главная / Проекты / Склад / Настройки**. Healthcheck: `GET /healthz`.

### HTTPS для камеры со смартфона

Браузеры дают доступ к камере (сканирование) только в защищённом контексте —
`localhost` или **HTTPS**. Для доступа со смартфона по локальной сети:

```bash
python -m scripts.make_cert          # самоподписанный сертификат с вашими IP → certs/
python main.py                       # при наличии certs/ поднимается HTTPS автоматически
```

Откройте на телефоне `https://<IP-ПК>:8000`, один раз примите предупреждение о
недоверенном сертификате. По обычному HTTP на сетевой IP камера работать не будет
(ручной ввод и USB/Bluetooth-сканер — работают и по HTTP).

> **Быстрый локальный запуск без PostgreSQL.** Задайте в `.env`
> `DATABASE_URL=sqlite:///./dev.db`, затем `alembic upgrade head` создаст схему в
> `dev.db`. Боевой режим рассчитан на PostgreSQL; `.env` и `*.db` в Git не попадают.

## Возможности (Этапы 1–3)

- **Склад:** двухуровневые категории; модели с весом/габаритами/ценой/упаковкой;
  количественный учёт с историей остатков и посерийный учёт с экземплярами,
  штрих-кодами, статусами (Активно/В ремонте/Списано) и историей статусов; фото
  моделей с оптимизацией; фильтры, сортировка (запоминается), архив; поиск по штрих-коду.
- **Проекты:** автонумерация, статусы Черновик/Забронирован/Завершён/Отменён,
  даты аренды (обе включительно), расчёт доступности и дефицита по пересекающимся
  бронированиям, подтверждение брони с дефицитом, копирование, архив.
- **Главная:** активные/просроченные брони, проекты с дефицитом, оборудование в ремонте.
- **Настройки компании:** реквизиты, VAT по умолчанию, часовой пояс.
